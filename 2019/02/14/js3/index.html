一、基本概念

简介和组成

简介： JavasScript是一门专为与网页交互而设计的脚本语言，由浏览器解析和执行，无需要编译。
组成： 一个完整的JavaScript由以下三个不同的部分组成

  • 核心（ECMAScript）：提供语言的核心功能
  • 文档对象模型（Document Object Model）DOM：提供访问和操作网页内容、结构的方法和接口
  • 浏览器对象模型（Browser Object Model）BOM：提供与浏览器交互的方法和接口

ECMAScript（核心）

WEB浏览器是ECMAScript的宿主环境之一，宿主环境提供基本的ECMAScript实现，同时也提供该言语的扩展【例如DOM】，方便语言和环境之间对接交互。其他宿主环境还包括Node、Adobe Flash等。
ECMAScript规定了JS以下的核心组成部分：语法、类型、语句、关键字、保留字、操作符、对象。

DOM

文档对象模型（DOM）是应用于HTML的应用程序编程接口（API），DOM把整个页面映射为一个多层次节点结构树，这些节点又包含着不同类型的数据。 而通过DOM创建的节点结构树，可以使开发人员轻松获得控制页面内容和结构的主动权，删除、添加、替换、修改任何节点。

DOM的级别

  • DOM1级：DOM1级由两个模块组成DOM核心和DOM HTML。DOM核心（DOM Core）规定的是如何映射基于XML的文档结构，以便简化对文档中任意部分的访问和操作。DOM HTML模块则是在DOM核心的基础上加以扩展，添加对HTML的对象和方法。
  • DOM2级：在原来DOM的基础上扩充了鼠标、用户界面事件、范围、遍历等模块，而且通过对象接口增加了对CSS的支持。
    • DOM视图（DOM View） - 定义了跟踪不同文档视图的接口。（例如应用CSS之前和之后的文档）
    • DOM事件（DOM Events） - 定义了事件和事件处理的接口。
    • DOM遍历和范围（DOM Traversal and Range）：定义了遍历和操作文档树的接口。
  • DOM3级：引入了以统一方式加载和保存文档的方法（在DOM加载和保存模块中定义）；新增了验证文档的方法（在DOM验证模块中定义）；DOM3级也对DOM核心进行了扩展，开始支持XML1.0，涉及XML infoset，Xpath和XML Base。

注意：DOM0级标准实际上是不存在的，它只作为DOM历史中的一个参照点，指的是IE 4.0和Netscape 4.0最初支持的DHTML

BOM

根本上来说，BOM处理浏览窗口和框架，但人们习惯把针对浏览器的JS扩展也算作BOM的一部分，以下就是这样的扩展：

  • 弹出新浏览窗口的功能
  • 移动、缩放和关闭浏览器的功能
  • 提供浏览器详细信息的navigator对象
  • 提供浏览器所加载页面的详细信息的Location对象
  • 提供用户显示器分辨率详细信息的screen对象
  • 对cookies的支持
  • 像XMLHttpRequest和IE的ActiveXObject这样的自定义对象

由于BOM没有标准可循，所以不同浏览器之间的实现可能会存在差误。

语法

任何语言的核心（JS里面的核心就是ECMASript）都会用来描述这门语言的基本工作原理。ECMAScript的语法大量借鉴了C及类C语言（JAVA和Perl）的语法。

  • 区分大小写。ECMAScript中的一切都区分大小写（变量、函数名、操作符）。
  • 标识符。就是指变量、函数、属性的名字，或者是函数的参数。以下是标识符的组合规则：
    • 首字母必须是字母、下划线或者美元符（$）
    • 其他字母可以是字母、下划线、美元符或者数字
  • 注释。有单选（//双斜杠），和多行注释（/ …. /）
  • 严格模式。ES5引入了严格模式的概念，严格模式为JavaScript定义了一种不同的解析和执行模型。引入严格模式的目的有：
    • 消除JavaScript语法的一些不合理、不严谨之处，减少一些怪异行为;
    • 消除代码运行的一些不安全之处，保证代码运行的安全；
    • 提高编译器效率，增加运行速度；
    • 为未来新版本的Javascript做好铺垫。
  • 语句。ECMAScript 中的语句以一个分号结尾；如果省略分号，则由解析器确定语句的结尾。虽然语句结尾的分号不是必须的，但好的编程习惯是任何时候都要加上。

标识符惯例是采用驼峰式书写方法，为了和ECMAScript内置的函数和对象命名方式一致，可以将其当作是一种最佳实践。

关键字和保留字

关键字

ECMA-262描述了一组具有特殊用途的关键字。关键字也是语言保留的，不能作用标识符。以下是ECMAscript的全部关键字：

       
break do instanceof typeof
case else new var
catch finally return void
continue for switch while
debugger* function this with
default if throw delete
in try    

保留字

保留字也不能用作标识符使用，它们将来可能被用来作为关键字使用。以下ECMAScript 3版定义的保留字：

       
abstract enum int short
boolean export interface static
byte extends long super
char final native synchronized
class float package throws
const goto private transient
debugger implements protected volatile
double import public

ECMAScript 5 把在非严格模式下运行时的保留字缩减为下列这些：

       
class enum extends super
const export import

在严格模式下，第 5 版还对以下保留字施加了限制：

     
implements package public
interface private static
let protected yield

二、变量、作用域及内存

1、变量基本概念

ECMAScript的变量是松散类型的，可以用来保存任何的数据类型。

命名

命名方法
变量名一般以类型前缀+有意义的单词组成，以驼峰式命名增加变量和函数名的可读性

前缀规范：
s：表示字符串。例如：sName，sHtml；
n：表示数字。例如：nPage，nTotal；
b：表示逻辑。例如：bChecked，bHasLogin；
a：表示数组。例如：aList，aGroup；
re：表示正则表达式。例如：rDomain，rEmail；
fn：表示函数。例如：fGetHtml，fInit；
o：表示以上未涉及到的其他对象，例如：oButton，oDate；
g：表示全局变量，例如：gUserName，gLoginTime；
i：整型

命名规则

  • 首字母为英文字母或者下划线
  • 然后组成 为，英文字母、数据和下划线
  • 禁止使用JavaScript的关键字和保留字
  • 严格区分大小写

声明

JavaScript使用关键字var声明一个变量。

不推荐的陋习写法：

  • 重复声明
  • 不声明直接赋值
  • 不要更改变量的类型，开始是值类型，后面就不要改为数值或别的
  • 不要隐式声明

推荐的做法：

  • 先声明，后读写 -【需要读写的变量，一定要先声明】
  • 先赋值，后运算 -【需要运算的变量，一定要先赋值】
  • 书写的时候，一般把变量写在页面的最上面，声明+定义，方便阅读和查找。
  • 任何情况下，都要使用var关键先声明变量。

如果声明了一个变量，但不对其进行赋值，该变量存在，其值为undefined。
在严格模式下，给一个没有声明的变量赋值是会报错的，非严格下虽然不会报错，但也可能会导致未知的BUG，建议始终使用var进行变量声明。

2、基本类型和引用类型的值

ECMASscript 变量可能包含两种不同数据类型的值：基本类型值和引用类型值。在给一个变量赋值时，解析器必须确定这个值是基本类型值还是引用类型值。

基本类型值：5种基本数据【number/string/boolean/undefined/null】是按值引用的，因为可以操作保存在变量中的实际值。有以下特点：

  • 占用空间固定，保存在栈中
  • 保存与复制的是值本身
  • 使用typeof检测数据类型
  • 基本数据类型是值类型

引用类型值：引用类型是保存在内存中的对象。JavaScript不允许直接操作对象的内存空间，换句话说，操作对象时实际操作的是对象的引用而不是实际的对象，为此，引用类型的值是按引用访问的。有以下特点：

  • 占用空间不固定，保存堆中
  • 保存与复制的是指向对象的一个指针
  • 使用instanceof检测数据的类型
  • 使用new()方法构造出来的对象是引用类型

可以用typeof操作符简单检测变量的数据类型，typeof操作符可能返回下列某个字条串：undefined/null/string/boolean/object/function

1、null返回的是“object”，因为特殊值null被认为是一个空对象的引用
2、因为typeof不是函数，只是一个操作符，所以它后面的括号不是必需的

动态的属性

对于引用类型的值，我们可以为其他添加属性和方法，也可以改变和删除其属性和方法。但我们不能为基本类型值添加属性和方法，尽管这样不会导致错误。

传递参数

ECMAScript中所有函数的参数都是按值传递的。就是说函数外部的值复制给函数内部的变量，就像把一个变量复制到另一个变量一样。在向参数传递值的时候，被传递的值会被复制给一个局部变量【即命名参数，或用ECMAScript的概念来说，是arguments对象中的一个元素】

检测类型

instanceof检测如果变量是给引用类型的实例，那么就返回true。例如：arr1 instanceof Array //变量arr1是Array吗？

3、执行环境和作用域

执行环境

全局执行环境是最外围的一个执行环境。在WEB浏览器中，全局执行环境被认为是window对象，因此所有全局变量和函数都是作为window对象的属性和方法创建的。
某执行环境中的所有代码执行完成之后，该执行环境被销毁，保存在其中的所有变量和函数定义也随之销毁【全局执行环境在应用程序结束时被销毁，例如关闭网页或者浏览器】

作用域

变量有全局变量和局部变量。
1、全局变量
包含：在函数体外定义的变量
在函数体内部定义的无var的变量
调用：可以在任何位置调用

2、局部变量
包含：在函数内部使用var定义的变量
函数的参数变量
调用：当前函数体内

3、优先级
局部变量高于全局变量
参数变量高于同名的全局变量

4、特性
忽略块级作用域
全局变量是全局对象的属性
局部变量是调用对象的属性
作用域链

  • 内层函数可以访问外层函数的局部变量
  • 外层函数不能访问内层函数的局部变量

生命周期

  • 全局变量：除非显示被删除，否则一直存在
  • 局部变量：自声明起至函数运动完毕或被显示删除
  • 回收机制：标记清除和引用计数
